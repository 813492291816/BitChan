BitChan Configuration

General Settings
{{form_settings.maintenance_mode.label(class_='heading-bold')}}
Prevent new posts from being made, display "Maintenance Mode" on all pages.
{{form_settings.debug_posts.label(class_='heading-bold')}}
This will show the entire database table entry for boards/threads/posts in an accordion. If kiosk is enabled, must be an Admin.
{{form_settings.theme.label(class_='heading-bold')}}
Sets the colors and CSS style for all pages.
{{form_settings.chan_update_display_number.label(class_='heading-bold')}}
Max number of Board/List updates on the home page.
{{form_settings.max_download_size.label(class_='heading-bold')}}
Max total size (MB) of post attachments to automatically download from external upload sites or torrent via i2p (0 disables auto-download).
{{form_settings.max_extract_size.label(class_='heading-bold')}}
Max size (MB) of uncompressed post attachments. If during extraction/decryption, the size surpasses this value, extraction/decryption will be cancelled. This setting also restricts the size of attachments when creating posts.
{{form_settings.always_allow_my_i2p_bittorrent_attachments.label(class_='heading-bold')}}
Always allow automatic downloading of I2P BitTorrent attachments originating from this BitChan instance.
{{form_settings.allow_net_file_size_check.label(class_='heading-bold')}}
Permit connecting (through tor) to check post attachment file size.
{{form_settings.allow_net_book_quote.label(class_='heading-bold')}}
Permit connecting (through tor) to get random book quotes for #stich and #godsong in posts.
{{form_settings.never_auto_download_unencrypted.label(class_='heading-bold')}}
If a post has unencrypted attachments from upload sites, always require the Download button to be pressed to download them.
{{form_settings.allow_unencrypted_encryption_option.label(class_='heading-bold')}}
Allow "Unencrypted" as an encryption option for post attachments.
{{form_settings.auto_dl_from_unknown_upload_sites.label(class_='heading-bold')}}
Automatically download from upload sites received in posts that don't already exist in your saved upload sites list.
{{form_settings.delete_sent_identity_msgs.label(class_='heading-bold')}}
This will automatically delete the sent messages in your identity mailboxes.
{{form_settings.post_timestamp.label(class_='heading-bold')}}
This will determine which timestamp to use to sort posts, threads, and boards when order is considered.
{{form_settings.post_timestamp_timezone.label(class_='heading-bold')}}
This will determine the timezone of the time that's shown in post headers. Warning: changing this from UTC is less secure because others can see which timezone you set if you enable Kiosk Mode or your share screenshots of posts with the timestamp visible. Changing this setting will cause all cached post HTML to be regenerated.
{{form_settings.post_timestamp_hour.label(class_='heading-bold')}}
This will determine whether 12-hour or 24-hour time is displayed in post headers. Changing this setting will cause all cached post HTML to be regenerated.
{{form_settings.title_text.label(class_='heading-bold')}}
Add custom text to the title to differentiate from other BitChan tabs in a browser.
{{form_settings.results_per_page_board.label(class_='heading-bold')}}
The number of threads to display per page on board pages.
{{form_settings.results_per_page_overboard.label(class_='heading-bold')}}
The number of threads to display per page on the overboard page.
{{form_settings.results_per_page_catalog.label(class_='heading-bold')}}
The number of threads to display per page on catalog pages.
{{form_settings.results_per_page_recent.label(class_='heading-bold')}}
The number of posts to display per page on the recent page.
{{form_settings.results_per_page_search.label(class_='heading-bold')}}
The number of results to display per page on the search page.
{{form_settings.results_per_page_mod_log.label(class_='heading-bold')}}
The number of results to display per page on the mod log page.
Home Page Message

Template HEAD HTML

Template BODY HTML
This is HTML that will be inserted in the layout page BODY.
I2P Torrent Tracker URLs
I2P torrent tracker list. Must be .i2p URLs separated by carriage returns (each URL on its own line).
Bitmessage Settings
{{form_settings.bm_connections_in_out.label(class_='heading-bold')}}
Set how bitmessage connects to peers. If desiring to use tor, keep in mind that you may need to initially set Outgoing Connections to use the clearnet to build up a list of hosts prior to setting it to use Tor, otherwise the bitmessage tor bootstrap address may take an incredibly long time to establish an initial connection.
{{form_settings.bitmessage_onion_services_only.label(class_='heading-bold')}}
Only allow Bitmessage to connect to onion services. Note: This is initially disabled because Bitmessage needs to first find connections before enabling. If this is enabled and Bitmessage has not found at least one onion address to connect to, it will never be able to connect to other peers.
Kiosk Settings
{{form_settings.enable_kiosk_mode.label(class_='heading-bold')}} (Kiosk User Management)
Enable Kiosk Mode.
{{form_settings.kiosk_login_to_view.label(class_='heading-bold')}}
Require users to log in to access BitChan.
{{form_settings.kiosk_allow_posting.label(class_='heading-bold')}}
Allow users to post on boards. Turn the kiosk into read-only by disabling this setting.
{{form_settings.kiosk_allow_pow.label(class_='heading-bold')}}
Allow users to perform additional proof of work (POW) for posts. Warning, this can allow DOS attacks on a kiosk.
{{form_settings.kiosk_allow_gpg.label(class_='heading-bold')}}
Allow users to encrypt or sign PGP messages in posts.
{{form_settings.kiosk_disable_bm_attach.label(class_='heading-bold')}}
Disallow users to use of the Bitmessage post Upload Method.
{{form_settings.kiosk_disable_i2p_torrent_attach.label(class_='heading-bold')}}
Disallow users to use of the I2P Torrent post Upload Method.
{{form_settings.kiosk_disable_torrent_file_download.label(class_='heading-bold')}}
Disallow users from downloading the torrent file of a post with an I2P/BitTorrent upload method. For the torrent file to be available, the post attachment option also needs to be set to unencrypted.
{{form_settings.kiosk_allow_download.label(class_='heading-bold')}}
Allow users to induce post attachment downloading with the Allow Download button.
{{form_settings.kiosk_ttl_option.label(class_='heading-bold')}}
Force posts to use a specific Time To Live (TTL) or allow the user to set the TTL with a maximum.
{{form_settings.kiosk_ttl_seconds.label(class_='heading-bold')}}
The custom TTL, in seconds, when forcing a specific TTL or setting a maximum TTL. Must be between 3600 and 2419200 seconds.
{{form_settings.kiosk_post_rate_limit.label(class_='heading-bold')}}
The refractory period (seconds) between posting.
{{form_settings.kiosk_max_post_size_bytes.label(class_='heading-bold')}}
The maximum allowable post size, in bytes. 0 disables, allowing the default max post size of {{config.BM_PAYLOAD_MAX_SIZE}} bytes.
{{form_settings.kiosk_attempts_login.label(class_='heading-bold')}}
The number of login attempts before a temporary login ban is made.
{{form_settings.kiosk_ban_login_sec.label(class_='heading-bold')}}
How many seconds to ban the user from logging in.
{{form_settings.kiosk_only_admin_access_mod_log.label(class_='heading-bold')}}
When kiosk mode is enabled, only Admins can view the Mod Log.
{{form_settings.kiosk_only_admin_access_search.label(class_='heading-bold')}}
When kiosk mode is enabled, only Admins can use the Search.
Security Settings
{{form_settings.enable_captcha.label(class_='heading-bold')}}
Require the completion of a captcha in order to post. Useful when Kiosk mode is enabled. Does not require session in browser.
{{form_settings.enable_verification.label(class_='heading-bold')}}
Enable DoS-protection. Makes the user initially wait 5 seconds then solve a captcha before being granted access. Useful when Kiosk Mode is enabled. Requires session in browser.
{{form_settings.hide_version.label(class_='heading-bold')}}
Hide the version number from the title and front page.
{{form_settings.enable_page_rate_limit.label(class_='heading-bold')}}
Enable rate-limiting page requests. Useful when Kiosk Mode is enabled. Requires session in browser.
{{form_settings.max_requests_per_period.label(class_='heading-bold')}}
When rate-limiting, the maximum number of requests per period before a user needs to reverify a session.
{{form_settings.rate_limit_period_seconds.label(class_='heading-bold')}}
When rate-limiting, the period (seconds) to allow the Maximum Requests. E.g. maximum of 10 requests per 60 seconds.
{{form_settings.remote_delete_action.label(class_='heading-bold')}}
When an admin delete of a post or thread is remotely performed by a board admin/mod, your local install can either delete or hide the post/thread.
{{form_settings.disable_downloading_upload_site.label(class_='heading-bold')}}
Disable downloading post attachments from upload sites.
{{form_settings.disable_downloading_i2p_torrent.label(class_='heading-bold')}}
Disable downloading post attachments from i2p torrents.
{{form_settings.ttl_seed_i2p_torrent_op_days.label(class_='heading-bold')}}
How long to seed i2p torrent post attachments for the original post of a thread. Value is in days.
{{form_settings.ttl_seed_i2p_torrent_reply_days.label(class_='heading-bold')}}
How long to seed i2p torrent post attachments for reply posts of a thread. Value is in days.
RSS Settings
{{form_settings.rss_enable.label(class_='heading-bold')}}
Enable Tor RSS feeds of posts to be generated for boards and threads. The main URLs of the RSS feed will use the specified Tor URL.
{{form_settings.rss_enable_i2p.label(class_='heading-bold')}}
Enable I2P RSS feeds of posts to be generated for boards and threads. The main URLs of the RSS feed will use the specified I2P URL.
{{form_settings.rss_url.label(class_='heading-bold')}}
The Tor URL to use when generating URLs in the Tor RSS feed. Note that if HTML is enabled for posts, the URLs will already be generated as relative URLs will not respect the domain listed in this setting. Disable HTML in RSS if you want the specified domain in the URLs to be used.
{{form_settings.rss_url_i2p.label(class_='heading-bold')}}
The I2P URL to use when generating URLs in the I2P RSS feed. Note that if HTML is enabled for posts, the URLs will already be generated as relative URLs will not respect the domain listed in this setting. Disable HTML in RSS if you want the specified domain in the URLs to be used.
{{form_settings.rss_number_posts.label(class_='heading-bold')}}
How many posts to include in each RSS file.
{{form_settings.rss_char_length.label(class_='heading-bold')}}
If HTML in posts is disabled, plain text posts will be truncated to this character limit.
{{form_settings.rss_use_html_posts.label(class_='heading-bold')}}
If available, generate RSS feed posts with popup HTML. HTML posts will already be truncated based on popup HTML settings. Otherwise, posts will be plain text truncated to the RSS character limit specified.
{{form_settings.rss_rate_limit_number_requests.label(class_='heading-bold')}}
How many RSS requests are allowed per time period.
{{form_settings.rss_rate_limit_period_sec.label(class_='heading-bold')}}
The time period (seconds) to rate limit RSS requests with a sliding window counter.
